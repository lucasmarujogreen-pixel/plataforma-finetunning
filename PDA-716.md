# [PDA-716](https://greenlegis.atlassian.net/browse/PDA-716) — POC de fine-tuning para extração de condições em análises normativas

## Descrição

**Objetivo**
Avaliar se o fine-tuning de um modelo é uma abordagem viável para automatizar ou apoiar a extração das “condições” em análises normativas, utilizando como base o histórico de análises já realizadas pelo time de consultoria.
**Contexto**
As “condições” extraídas durante a análise normativa são fundamentais para definir a aplicabilidade da norma aos perfis de clientes da Greenlegis. Atualmente, essa interpretação depende do time de consultoria, que avalia a norma e define os critérios de vínculo.
Esta POC tem como objetivo testar se um modelo treinado com exemplos reais de normas e suas respectivas análises consegue aprender o padrão de extração das condições e gerar resultados consistentes para novas normas.
**Escopo da POC**
- Levantar uma base de exemplos contendo norma, análise realizada e condições extraídas.
- Definir o formato ideal do dataset para treinamento.
- Preparar os dados para fine-tuning, incluindo limpeza, padronização e separação entre treino, validação e teste.
- Treinar um modelo com foco na geração das condições da análise.
- Executar testes com normas não utilizadas no treinamento.
- Comparar os resultados do modelo fine-tuned com as análises humanas existentes.
- Avaliar custo, esforço, manutenção e escalabilidade da abordagem.
**Entregáveis**
- Dataset estruturado para fine-tuning.
- Modelo treinado ou experimento executado com fine-tuning.
- Resultado da POC com amostra de normas testadas.
- Comparativo entre condições geradas pelo modelo e condições cadastradas manualmente.
- Análise de custo, esforço e manutenção.
- Recomendação sobre continuidade ou descarte da abordagem.
**Critérios de aceite**
- Deve existir uma base mínima de dados preparada para treinamento.
- O modelo deve ser testado com normas fora da base de treino.
- Os resultados devem ser comparados com análises humanas.
- A POC deve indicar se fine-tuning entrega ganho relevante em relação ao enriquecimento de prompt.
- A decisão final deve considerar qualidade da resposta, custo, esforço de atualização e necessidade de revisão humana.
